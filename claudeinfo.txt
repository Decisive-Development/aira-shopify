## Code Standards

- **Tailwind v4** — use Tailwind classes for everything. Only use inline `style=` when Tailwind cannot express the value (e.g. custom gradients, box-shadow).
- **No opacity modifiers** on text/colors unless explicitly requested (`text-main/65` etc.)
- **No inline styles** for colors — use Tailwind tokens (`text-main`, `bg-secondary`) instead of `style="color: var(--color-primary)"`.
- **Indentation** — tabs, 4-space width.
- **Liquid formatting** — use `{%- -%}` (whitespace control) on Liquid tags inside HTML.
- **Modular** — reusable UI elements go in `snippets/`. Sections use `{% render %}` to pull them in.
- **CMS-first** — expose content/settings via `{% schema %}`. Hardcode structure, make content editable.
- **No backwards-compat hacks** — delete unused code, don't comment it out.
- **Don't push** — user pushes to Shopify manually. Never auto-push.

## Design Patterns

### Floating pill
Both the nav and footer use a floating pill aesthetic:
- `rounded-2xl` corners (NOT `rounded-full`)
- `bg-secondary` background
- `box-shadow: 0 12px 40px rgba(10,26,52,0.35), 0 4px 16px rgba(255,255,255,0.15)`
- Max width 1280px (`max-w-7xl`), centered

### CTA Buttons
Always use `snippets/cta-button.liquid` via `{% render 'cta-button', ... %}`.

Parameters:
| Param         | Values                            | Default    |
|---------------|-----------------------------------|------------|
| `label`       | String (required)                 | —          |
| `url`         | String                            | `#`        |
| `variant`     | `primary` / `accent` / `ghost` / `inverse` / `blue` | `primary`  |
| `size`        | `sm` / `md` / `lg`               | `md`       |
| `element`     | `a` / `button`                    | `a`        |
| `button_type` | `button` / `submit` / `reset`     | `button`   |
| `extra_classes` | String of Tailwind classes      | —          |

Sizes: `sm` = `px-6 py-2.5`, `md` = `px-8 py-3`, `lg` = `px-10 py-4`
Rounding: always `rounded-xl` (soft, not pill)
Variants: `primary` = bg-main/text-secondary, `accent` = bg-highlight/text-white, `ghost` = border-2/border-main/text-main, `inverse` = bg-secondary/text-main, `blue` = bg-highlight-blue/text-white

Default size is `md`. Only specify `size` when intentionally deviating.

## Sections Built

### sections/header.liquid
- Two-state fixed nav, `top-0`, z-50
- **Transparent state** (at page top): white logo (`aira-logo-text-white.svg`), white nav text, white icons, no background — overlays hero content
- **Solid state** (on scroll): floating pill with `bg-secondary`, rounded-2xl, shadow — blue logo (`aira-logo-text-blue.svg`), `text-main` nav/icons
- Scroll threshold: 50px. Transparent fades out (opacity 0.3s), solid slides down (translateY 0.4s cubic-bezier). Reverse on scroll to top.
- If page loads already scrolled, `no-transition` class prevents flash
- CTA: `cta-button` snippet, `variant: accent` (both states)
- **Mobile**: icon-only logo (`aira-logo-only-white.svg` / `aira-logo-only-dark.svg`) via `md:hidden` / `hidden md:block`. Hamburger toggle → animated dropdown (CSS transitions: opacity + translateY, 0.25s ease). Menu text `text-lg`, child links `text-base`. CTA `size: 'sm'` for thinner padding. Uses `menu-closed` / `menu-open` classes with `toggleMenu()` JS.
- **Product pages**: transparent nav gets `bg-main` (blue) background instead of transparent, no `pt-2` offset. Conditional via `request.page_type == 'product'`.
- Cart icon links have `data-cart-toggle` attribute — cart drawer JS binds to these.
- Cart count badges: `data-cart-count-bubble` (bubble span) + `data-cart-count` (count text) updated dynamically by cart drawer JS.
- Spacer div (`h-20`) in `layout/theme.liquid` prevents content hiding behind fixed nav
- JS: inline `<script>`, adds/removes `.scrolled` class on `[data-header]`

### sections/hero.liquid
- Full-width, full-cover image hero. Foreground image fills entire section as background (`absolute inset-0 object-cover`, 140% height for parallax headroom).
- Three darken layers: uniform `bg-black/35`, bottom gradient `from-black/60` for copy contrast, top gradient `h-40 from-black/50` for navbar contrast.
- Parallax: JS translates image layer on scroll (35% rate, rAF-driven, respects `prefers-reduced-motion`).
- Copy positioned bottom-left (`justify-end`), capped at `max-w-2xl`.
- CMS controls: eyebrow, heading, subheading, primary + secondary CTA, foreground image, background image, overlay, full-height toggle, container width, vertical padding.
- Background: night-sky gradient toggle (default on) — fallback when no image set.
- Primary CTA: `cta-button` snippet, `variant: accent`, `size: lg`.
- Unsplash placeholder when no foreground image (theme-check-disable applied).

### sections/footer.liquid
- Outer `<footer>` has blue→burnt-orange sunset gradient background: `linear-gradient(to bottom, #0A1A34 50%, ... #6b2f00 100%)`
- Sunset gradient widened: `ellipse 140% 40%` for broader coverage
- Floating pill inside: `bg-secondary`, `rounded-2xl`, wide vertical padding (`py-12 md:py-14`)
- Grid: `2fr 1fr 1fr 1fr` — left col wider (logo + newsletter + socials), 3 right cols equal (Shop / Resources / Company)
- **Mobile**: first two columns side-by-side (`grid-cols-2`), brand col `col-span-2`, company col `col-span-2`. Email form full-width (`flex flex-col sm:flex-row`, input `w-full sm:w-70`, button `w-full sm:w-auto`).
- Newsletter form: Shopify `form 'customer'` with `contact[tags]: newsletter`. Input `rounded-xl`, fixed width. Subscribe uses `cta-button` snippet (`element: button`, `button_type: submit`, `variant: primary`)
- Social icons: Instagram, YouTube, TikTok — CMS-controlled URLs via schema (hidden if blank)
- Copyright bar: below the pill, `text-white`, `© 2026 Aira Wellness. · Privacy Policy · Terms of Service`
- Links: hardcoded paths. Social URLs: CMS via schema.

### snippets/product-card.liquid
Reusable product card for collection/shop pages. Used by `sections/collection.liquid`. Pass via `{% render 'product-card', product: product, badge: 'Best Seller', description: '...' %}`.
- Portrait image `aspect-4/5` with hover scale-105. Optional badge pill top-left.
- Title is plain `<h3>` (not a link — the CTA is the action).
- Optional `description` param from CMS; falls back to `product.description | strip_html | truncate: 120`.
- Colour option (name `Colour`/`Color`) → circle swatches with inline `style="background: #hex"`. Known colours: Black=`#1A1A1A`, Beige=`#C8B89A`.
- All other options → pill buttons.
- Selling plan (subscription) support: if product has selling plans, shows subscribe/one-time toggle with radio buttons, pricing, save amount. Subscribe promoted first, one-time below with "or" divider.

- Variant data embedded as `<script type="application/json" data-product-variants>` including `selling_plan_allocations` for JS.
- `.product-card-description` CSS in `critical.source.css` handles list/heading styles for Shopify rich-text HTML.

### sections/before-after-slider.liquid
Drag comparison slider. Section heading (via `section-heading` snippet) above a `max-w-2xl` (672px, reduced ~15% from original 768px) comparison widget.
- After image (`w-full h-auto block`) is the base layer, determines container height.
- Before pane: `absolute inset-0`, `clip-path: inset(0 R% 0 0)` on the pane div — clips image + label together.
- Drag handle: `w-10` container, centred via `left: pct%; margin-left: -20px` (no transform). White circle with ←→ arrows SVG, white 2px vertical line.
- Cursor: `cursor-grab` at rest, `cursor-grabbing` while dragging.
- Labels: `bg-main text-secondary` pill in bottom corners, clipped with the before pane.
- Default images: `assets/images/With_Compressed.webp` (after, 600×654) and `assets/images/Without_Compressed.webp` (before, 600×654). CMS image_picker overrides these.
- JS: mouse + touch drag; `touchmove` uses `passive: false` to prevent scroll during drag. No syncImageWidth needed.
- CMS: eyebrow, heading, subheading, heading_align, heading_size, before_image, before_label, after_image, after_label, container_width, padding_vertical.

### sections/health-effects.liquid
Grid layout (50/50 on lg): focus panel left, 2×2 card grid right. Heading centered.
- **Focus panel** (left): image `flex-1`, solid `bg-main` block at bottom with title + description + optional link. No icon. Stacked layers per block, crossfade via opacity (500ms).
- **Card grid** (right): 2×2 `aspect-square` cards. Each card uses its block image as full background with title in overlay div at bottom (`bg-linear-to-t from-black/70 via-black/40 to-transparent`). Icons removed.
- Active card hidden from grid. "Next" card gets `@property --border-progress` conic-gradient sweep animation (5s, clockwise from 12 o'clock, orange #FF4000).
- **Hover** = preview (shows content in focus panel, grid unchanged). **Click** = lock (swaps grid positions). Prevents hover-spam bug.
- Newly appearing cards animate in (scale 0.92→1, opacity 0→1, 350ms).
- Auto-rotate timer (5s `setTimeout`). Pauses on grid hover (sweep animation pauses too via `animation-play-state`).
- Scroll-in animation via IntersectionObserver.
- Blocks: `effect` type — icon, title, subtitle, image_picker, richtext description, link_label, link_url.
- Defaults: 5 blocks (High blood pressure, Snoring, Dry mouth, Poor sleep quality, Facial development).

### sections/benefits-slider.liquid
Horizontally scrollable card slider for product benefits. Cards are `w-72 flex-shrink-0`, image top (`aspect-[3/4]`), title + description below.
- Track left padding uses `max(1.5rem, calc((100vw - 1280px) / 2 + 3rem))` to align first card with container edge on all screen widths.
- Prev/next arrow buttons auto-disable at scroll boundaries. JS scrolls by `cardWidth + 20px gap`.
- Scrollbar hidden via inline `<style>` targeting `[data-benefits-track]`.
- Scroll-in animations: IntersectionObserver, `data-benefits-animate`, removes `opacity-0 translate-y-6`.
- Separate `padding_top` / `padding_bottom` controls (not a single padding_vertical).
- CMS: eyebrow, heading, subheading, padding_top, padding_bottom. Blocks: `slide` type with image_picker, heading, description.

### sections/social-proof.liquid
Full-width problem-vs-solution stats section over a background image.
- **Background**: CMS image_picker, `bg-black/35` darken layer + radial vignette (`radial-gradient ellipse, transparent 40% → rgba(0,0,0,0.75) 100%`). Fallback: deep navy solid color.
- **Heading**: centered at top — eyebrow, heading (reduced `text-3xl md:text-4xl lg:text-5xl`), subheading (hidden on mobile).
- **Two block types**: `problem` (left column, red/orange `text-highlight`) and `benefit` (right column, bright green `#4ADE80`).
- **Block fields**: `value` (stat number), `keyword` (bold emphasis word e.g. "Snoring"), `label` (description), `footnote` (source), `footnote_url` (links to study). Footnotes colored `--color-tertiary-light` (#9BB7D4), clickable when URL provided.
- **Desktop layout** (`lg+`): two-column grid, full-width (not container-constrained), `max-w-[50vw]` per column. Left = problems left-aligned, right = benefits right-aligned. Vertical divider `border-l border-white/10`. Each stat row: large value → keyword → label → source link. Scroll-in animations via IntersectionObserver.
- **Mobile layout** (`< lg`): separate code block (`lg:hidden`). Paired rows — each row is `grid-cols-2` with problem on left, matching benefit on right. Compact: no footnotes, smaller text, tighter spacing. Uses counter-based iteration to find nth problem/benefit per row (Liquid can't index arrays with string keys from `split`).
- **Stats are research-backed**: all cite peer-reviewed studies (PubMed, CDC, journal publications). No "Aira survey" claims. Problems: snoring prevalence, sleep deprivation, heart disease risk, dry mouth. Benefits: snoring reduction, airway resistance, cardiovascular health, nasal obstruction — paired to mirror the problems.
- CMS: eyebrow, heading, subheading, left_heading, right_heading, background_color, background_image.

### sections/judgeme-reviews.liquid
- Judge.me review widget section. Section heading (via `section-heading` snippet, align center).
- CMS `product` picker to select which product's reviews to display.
- Renders Judge.me `jdgm-review-widget` div with product data attributes. Supports legacy widget fallback.
- Injects `jdgm.data.reviewWidget` script block if metafield data available.
- **CSS overrides** injected via JS (polling for `jm-` elements, then appending `<style>` to `<head>`). All widget selectors prefixed with `#judgeme_product_reviews` for specificity. Hardcoded hex values (not CSS vars) since injected via JS string concatenation.
- **Key styling decisions:**
  - Stars: blue `#087CA7` (not default green), via `.jm-star-rating__font-icon` + `--jm-review-widget-star-and-histogram-color` variable override.
  - Review cards: `#FAFAFA` bg, `#F2F2F2` border, `rounded-2xl`, hover border darkens.
  - Reviewer avatar (initial square): hidden via `display: none`. Just name + date shown.
  - Review images: positioned on the right of the card (not below) via JS that forces `flex-direction: row` on `.jm-review-item > .jm-stack`. Thumbnails: 140px wide, 3:4 aspect ratio, rounded.
  - Floating media gallery (`.jm-media-preview`): hidden — images only show in their review card.
  - Empty `.jm-cluster` divs inside reviews: hidden via JS.
  - CTA button (`.jm-action-buttons__button`): blue `#087CA7`, branded font, `inline-size: 60%` via `.jm-review-widget-minimal-header__action-buttons .jm-button`.
  - Primary/secondary buttons: branded fonts, ghost/outline style, navy hover fill.
  - Pagination: no borders on arrow buttons.
- **Write-review modal** (uses `jdgm-` prefixed classes, not `jm-`):
  - Product image: 160×290px tall (not square), `object-fit: cover`, rounded.
  - Stars: blue `#087CA7`.
  - Form inputs: rounded borders, blue focus ring.
  - Nav buttons: Next/Close = blue `#087CA7`, Back = ghost/outline.
  - Upload area: dashed border, blue hover highlight.
  - GDPR text: subtle, links in blue.
- **Important:** Judge.me v2 widget uses `jm-` prefixed classes for the review list, but `jdgm-` prefixed classes for the write-review modal. Both need separate CSS targeting.
- **Important:** Judge.me's `.jm-cluster` layout uses CSS custom properties (`--cluster-wrap`) set inline. CSS `!important` alone can't override inline styles on variables — requires JS `el.style.setProperty()` to force overrides.
- CMS: eyebrow, heading, subheading, heading_size, product, background_color, padding_vertical.
