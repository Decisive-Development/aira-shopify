## Code Standards

- **Tailwind v4** — use Tailwind classes for everything. Only use inline `style=` when Tailwind cannot express the value (e.g. custom gradients, box-shadow).
- **No opacity modifiers** on text/colors unless explicitly requested (`text-main/65` etc.)
- **No inline styles** for colors — use Tailwind tokens (`text-main`, `bg-secondary`) instead of `style="color: var(--color-primary)"`.
- **Indentation** — tabs, 4-space width.
- **Liquid formatting** — use `{%- -%}` (whitespace control) on Liquid tags inside HTML.
- **Modular** — reusable UI elements go in `snippets/`. Sections use `{% render %}` to pull them in.
- **CMS-first** — expose content/settings via `{% schema %}`. Hardcode structure, make content editable.
- **No backwards-compat hacks** — delete unused code, don't comment it out.
- **Don't push** — user pushes to Shopify manually. Never auto-push.

## Design Patterns

### Floating pill
Both the nav and footer use a floating pill aesthetic:
- `rounded-2xl` corners (NOT `rounded-full`)
- `bg-secondary` background
- `box-shadow: 0 12px 40px rgba(10,26,52,0.35), 0 4px 16px rgba(255,255,255,0.15)`
- Max width 1280px (`max-w-7xl`), centered

### CTA Buttons
Always use `snippets/cta-button.liquid` via `{% render 'cta-button', ... %}`.

Parameters:
| Param         | Values                            | Default    |
|---------------|-----------------------------------|------------|
| `label`       | String (required)                 | —          |
| `url`         | String                            | `#`        |
| `variant`     | `primary` / `accent` / `ghost` / `ghost-light` / `inverse` / `blue` | `primary`  |
| `size`        | `sm` / `md` / `lg`               | `md`       |
| `element`     | `a` / `button`                    | `a`        |
| `button_type` | `button` / `submit` / `reset`     | `button`   |
| `extra_classes` | String of Tailwind classes      | —          |

Sizes: `sm` = `px-6 py-2.5`, `md` = `px-8 py-3`, `lg` = `px-10 py-4`
Rounding: always `rounded-xl` (soft, not pill)
Variants: `primary` = bg-main/text-secondary, `accent` = bg-highlight/text-white, `ghost` = border-2/border-main/text-main, `ghost-light` = border-secondary/50/text-secondary/hover-bg-secondary/10 (cream outline on dark surfaces), `inverse` = bg-secondary/text-main, `blue` = bg-highlight-blue/text-white

Default size is `md`. Only specify `size` when intentionally deviating.

## Sections Built

### sections/header.liquid
- Two-state fixed nav, `top-0`, z-50
- **Transparent state** (at page top): white logo (`aira-logo-text-white.svg`), white nav text, white icons, no background — overlays hero content
- **Solid state** (on scroll): floating pill with `bg-secondary`, rounded-2xl, shadow — blue logo (`aira-logo-text-blue.svg`), `text-main` nav/icons
- Scroll threshold: 50px. Transparent fades out (opacity 0.3s), solid slides down (translateY 0.4s cubic-bezier). Reverse on scroll to top.
- If page loads already scrolled, `no-transition` class prevents flash
- CTA: `cta-button` snippet, `variant: accent` (both states)
- **Mobile**: icon-only logo (`aira-logo-only-white.svg` / `aira-logo-only-dark.svg`) via `md:hidden` / `hidden md:block`. Hamburger toggle → animated dropdown (CSS transitions: opacity + translateY, 0.25s ease). Menu text `text-lg`, child links `text-base`. CTA `size: 'sm'` for thinner padding. Uses `menu-closed` / `menu-open` classes with `toggleMenu()` JS.
- **Product pages**: transparent nav gets `bg-main` (blue) background instead of transparent, no `pt-2` offset. Conditional via `request.page_type == 'product'`.
- Cart icon links have `data-cart-toggle` attribute — cart drawer JS binds to these.
- Cart count badges: `data-cart-count-bubble` (bubble span) + `data-cart-count` (count text) updated dynamically by cart drawer JS.
- Spacer div (`h-20`) in `layout/theme.liquid` prevents content hiding behind fixed nav
- JS: inline `<script>`, adds/removes `.scrolled` class on `[data-header]`

### sections/hero.liquid

### sections/health-effects.liquid
Grid layout (50/50 on lg): focus panel left, 2×2 card grid right. Heading centered.
- **Focus panel** (left): image `flex-1`, solid `bg-main` block at bottom with title + description + optional link. No icon. Stacked layers per block, crossfade via opacity (500ms). Mobile: tighter padding (`px-6 py-5`), smaller title (`text-lg`) + body (`text-sm`) so the image gets more room.
- **Card grid** (right): 2×2 `aspect-square` cards. Each card uses its block image as full background with title in overlay div at bottom (`bg-linear-to-t from-black/70 via-black/40 to-transparent`). Icons removed.
- **Initial state**: Liquid loop finds the block whose title contains "snoring" (case-insensitive) and uses that as the initial focus + hides its card from the grid. Falls back to index 0 if no match.
- **Interaction model**: manual-only. Hover = preview (shows content in focus panel, grid unchanged). Click = lock (swaps grid positions). No auto-rotate timer, no conic-gradient sweep, no `is-next` indicator — those were removed in Session 26.
- Newly appearing cards animate in (scale 0.92→1, opacity 0→1, 350ms).
- Scroll-in animation via IntersectionObserver.
- Blocks: `effect` type — icon, title, subtitle, image_picker, richtext description, link_label, link_url.
- Defaults: 5 blocks (High blood pressure, Snoring, Dry mouth, Poor sleep quality, Facial development).

### sections/social-proof.liquid
Full-width problem-vs-solution stats section over a background image.
- **Background**: CMS image_picker, `bg-black/35` darken layer + radial vignette (`radial-gradient ellipse, transparent 40% → rgba(0,0,0,0.75) 100%`). Fallback: deep navy solid color.
- **Heading**: centered at top — eyebrow, heading (reduced `text-3xl md:text-4xl lg:text-5xl`), subheading (hidden on mobile).
- **Two block types**: `problem` (left column, red/orange `text-highlight`) and `benefit` (right column, bright green `#4ADE80`).
- **Block fields**: `value` (stat number), `keyword` (bold emphasis word e.g. "Snoring"), `label` (description), `footnote` (source), `footnote_url` (links to study). Footnotes colored `--color-tertiary-light` (#9BB7D4), clickable when URL provided.
- **Desktop layout** (`lg+`): two-column grid, full-width (not container-constrained), `max-w-[50vw]` per column. Left = problems left-aligned, right = benefits right-aligned. Vertical divider `border-l border-white/10`. Each stat row: large value → keyword → label → source link. Scroll-in animations via IntersectionObserver.
- **Mobile layout** (`< lg`): separate code block (`lg:hidden`). Paired rows — each row is `grid-cols-2` with problem on left, matching benefit on right. Compact: no footnotes, smaller text, tighter spacing. Uses counter-based iteration to find nth problem/benefit per row (Liquid can't index arrays with string keys from `split`).
- **Stats are research-backed**: all cite peer-reviewed studies (PubMed, CDC, journal publications). No "Aira survey" claims. Problems: snoring prevalence, sleep deprivation, heart disease risk, dry mouth. Benefits: snoring reduction, airway resistance, cardiovascular health, nasal obstruction — paired to mirror the problems.
- CMS: eyebrow, heading, subheading, left_heading, right_heading, background_color, background_image.

### Session 26 — Why Aira animated rebuild + Health Effects manual-only (3 May 2026)

**Why Aira section rebuild (sections/benefits-grid.liquid) — full replacement:**

Iterated through four design briefs to land on the final approach. Replaced the static 2-col image+text grid with a pinned full-viewport scroll experience.

- **Final architecture: CSS sticky + GSAP animations** (no GSAP ScrollTrigger pin).
- A tall `.why-aira-wrapper` div provides scroll distance (`height: calc(N * 100vh)` where N = block count). A `.why-aira-sticky` child with `position: sticky; top: 0; height: 100vh` pins to the viewport while the user scrolls through the wrapper.
- A vanilla scroll listener on `window` calculates `progress = -wrapper.getBoundingClientRect().top / (wrapperH - viewH)`. Pure math on `getBoundingClientRect` — no caching, no DOM mutation, works on resize/refresh/monitor switch.
- GSAP is loaded from CDN (`cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js`, defer) and used **only** for slide-in/out timeline animations — no ScrollTrigger plugin, no pin spacers.
- Each slide is `position: absolute; inset: 0` stacked in the sticky container. Active slide has `opacity: 1`, others fade.
- Animation choreography per slide: eyebrow (slide 1 only) fade in, gradient number (`80px–140px clamp`, navy→teal `-webkit-background-clip: text` gradient) scale+fade, orange accent line grows from 0 to 56px, heading reveals **word by word** (each word wrapped in `.word > .word-inner` spans, JS splits text on whitespace, GSAP staggers 50ms), body fades up, product image scales 0.92→1.0.
- Out animation: words slide up out of frame (`y: -110%`), elements fade and shift up.
- Right-edge progress indicator: dots (one per block) connected by progress lines whose orange fill animates per-segment based on overall scroll progress.
- Background decorative circle: navy/teal radial gradient at 6% alpha, drifts on infinite GSAP yoyo tween (no scroll dependency — earlier ScrollTrigger version conflicted with the pin trigger).
- Dot click → smooth-scrolls to that slide's midpoint progress.
- Mobile: switches to single-column, image moves above text (`order: -1`), text-aligns center, padding reduced.
- Schema: section settings (eyebrow, default product image), `benefit` blocks (heading, richtext body, optional per-slide image override).
- Section background: `bg-bg-soft` so adjacent page sections don't bleed through during the pin.

**Iteration history (kept here for context — final form is the V3 above):**
- V1: Sticky image + scrolling benefits (CSS sticky on a 40% column, IntersectionObserver fade-in on benefit blocks). Replaced because the brief escalated.
- V2: GSAP ScrollTrigger `pin: true` with manual scroll-space wrapper. Pin worked but: container had no background (bleed-through from sections below), grid columns rendered 560px instead of 1fr, conflicting bg-circle ScrollTrigger, broken on resize/mid-scroll refresh. Patched four bugs but the approach was structurally fragile.
- V3 (final): Abandoned ScrollTrigger pin entirely. CSS `position: sticky` for pinning + plain scroll listener for progress + GSAP only for animations. Battle-tested pattern (used by Apple/Stripe/Linear).

**Health Effects manual-only (sections/health-effects.liquid):**

- Removed auto-rotate timer (`ROTATE_MS`, `setTimeout`, `rotateNext`, `resetTimer`, `paused` flag, grid hover pause/resume listeners).
- Removed conic-gradient sweep visual (`@property --border-progress`, `@keyframes he-sweep`, `.he-card.is-next`, `.sweep-paused`) — those existed only as a countdown indicator for auto-advance.
- Initial focus card now defaults to "Snoring": Liquid loop at the top of the section iterates blocks, sets `assign initial_index = forloop.index0` when `block.settings.title | downcase` contains `'snoring'`, falls back to 0 if no match.
- Both the focus panel `opacity-100` and the grid `hidden` class are gated on `forloop.index0 == initial_index` (was `forloop.first`).
- JS `lockedIndex` initialised from Liquid: `var lockedIndex = {{ initial_index }};`.
- Kept: hover-preview, click-lock, card-entering scale animation when a card reappears in the grid, IntersectionObserver scroll-in.

**Health Effects focus panel mobile sizing:**
- Padding `p-8 lg:p-10` → `px-6 py-5 lg:p-10` (tighter horizontal + much shorter vertical on mobile).
- Title `text-2xl lg:text-3xl` → `text-lg lg:text-3xl`.
- Body description `text-base` → `text-sm lg:text-base`.
- Gap `gap-3` → `gap-2 lg:gap-3`.
- Net effect: more of the focus image is visible on mobile because the navy text block takes less vertical space.

**Key learnings:**
- GSAP ScrollTrigger `pin: true` is fundamentally fragile for full-viewport pinned sections — it mutates the DOM (creates pin-spacers), depends on cached pixel measurements, and breaks on resize / mid-scroll refresh / monitor switch. Three rounds of patches (`invalidateOnRefresh`, manual resize handlers, `addEventListener('refresh', ...)`) couldn't fully fix the edge cases. The CSS `position: sticky` + manual scroll-listener pattern is what production sites (Apple, Stripe, Linear) actually use.
- When you need GSAP for animation quality but want robust pinning, separate the concerns: CSS for pinning, scroll listener for progress math, GSAP only for the visual animations themselves.
- Auto-rotate carousels with countdown indicators feel pushy on educational/health content — manual-only with hover-preview reads as "explore at your own pace" and is what users actually want when they're absorbing information rather than browsing.
- Liquid's `forloop.index0` + a CMS-content-aware initial state (matching by title substring) is more flexible than hardcoding "the first block is special" — the merchant can reorder blocks freely without breaking the default focus.

- For configurable per-step CTA styling, a `select` schema setting with three named variants (`link` / `button` / `button-blue`) is cleaner than per-instance Tailwind classes in JSON. Render branches via Liquid `if`/`elsif`/`else`. The schema option labels educate the merchant on visual intent ("Orange button (matches nav)") rather than bg-class names.
- Tailwind v4 arbitrary variants like `[&_p+p]:mt-1.5` let you style richtext-rendered HTML inside a wrapper div without forking schemas or building a custom prose component. Pairs well with Shopify's `richtext` setting type.
- Two-grid "row of images on top, row of captions below" layouts ALWAYS break on mobile when both grids collapse to single columns. Default to single-grid + per-cell `flex flex-col` instead — works at every viewport without media-query gymnastics.
- For desktop image-row alignment when captions vary in line count: `items-stretch` on the grid (default) + `flex flex-col h-full` on each cell + `grow` on the variable-height text wrapper + `mt-auto` on the fixed-position element (image). No JS, no row-template tricks.
- Page-level JS belongs in the LAST section of the page rather than a global asset — it's auto-scoped via element existence checks (`if (els.length)`), gets shipped only when the page renders, and stays alongside the markup it operates on.
- For inline emphasis spans in headlines, the two-field pattern (`headline` + `headline_accent` + Liquid `replace`) is reusable across hero variants. Used by Our Story hero, Our Story closing, and now How it works hero — all three share the same liquid block at the top of the section.
- Rebuild Tailwind (`npm run build:css`) after every CSS edit to `critical.source.css`. The compiled `critical.css` ships, the source file does not (it's `.shopifyignore`d).
