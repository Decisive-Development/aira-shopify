# Aira Shopify — Terminal Commands Reference

Store: aira-breathing
Theme: Aira-theme


## Deploying to Shopify

# Push all local changes to the live theme
shopify theme push --theme Aira-theme -s aira-breathing

# Push a single file
shopify theme push --theme Aira-theme -s aira-breathing --only sections/faq.liquid

# Push matching files (e.g. all sections)
shopify theme push --theme Aira-theme -s aira-breathing --only "sections/*.liquid"

# Pull remote theme down to local (e.g. after editing in the Shopify theme editor)
shopify theme pull --theme Aira-theme -s aira-breathing

# List all themes on the store
shopify theme list -s aira-breathing

IMPORTANT — when adding a NEW section that's referenced in a template JSON:
  1. Push the section file first:   --only sections/new-section.liquid
  2. Then push the template:        --only templates/index.json
  Shopify validates JSON references on upload. If both go in one batch the JSON
  can arrive before the section file, causing a "does not refer to an existing
  section file" error.

Note: .shopifyignore controls which files are excluded from push/pull/dev.


## Development (Live Preview)

1. Open two terminals
2. Terminal 1:  npm run watch:css     (watches Tailwind source, recompiles on change)
3. Terminal 2:  npm run dev           (starts Shopify dev server + theme editor sync)
4. Edit files — CSS auto-compiles, Shopify dev server auto-reloads in the browser

npm run dev              Shopify dev server with live reload
                         Runs: shopify theme dev --theme Aira-theme --theme-editor-sync -s aira-breathing
                         Opens a local preview URL (usually localhost:9292)

npm run watch:css        Watch & auto-compile Tailwind CSS
                         Runs: @tailwindcss/cli -i ./assets/critical.source.css -o ./assets/critical.css --watch
                         Must run in a separate terminal alongside npm run dev

npm run build:css        One-off Tailwind CSS compile (no watch)
                         Run before deploying if watch:css wasn't running
                         Ensures assets/critical.css includes all used Tailwind classes


## Git

git status                   See what's changed
git add <file>               Stage a file for commit
git commit -m "message"      Commit staged changes
git push origin main         Push commits to GitHub
git log --oneline -10        View last 10 commits


## File Structure Quick Reference

assets/critical.source.css    Tailwind source — edit this file
assets/critical.css           Compiled output — don't edit, auto-generated by build:css
sections/*.liquid             Shopify section files
snippets/*.liquid             Reusable components (cta-button, section-heading, etc.)
templates/index.json          Homepage section order + settings
layout/theme.liquid           Main layout wrapper
config/settings_data.json     Theme settings (auto-generated by Shopify editor)
.shopifyignore                Files excluded from Shopify CLI operations
claudeinfo.txt                Project reference doc (ignored by Shopify)
commands.txt                  This file (ignored by Shopify)
